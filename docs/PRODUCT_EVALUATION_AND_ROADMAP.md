# ライフイベント管理サービス — プロダクト評価・UI/UX・AI 提案

## 1. 現状サマリー（コードベースの把握）

### 1.1 アプリ構成
- **スタック**: Expo (React Native) + TypeScript + Supabase
- **ナビ**: タブ4本（ホーム / カレンダー / 年表 / 外部サービス）
- **主要画面**: スプラッシュ → オンボーディング → ホーム、イベント一覧（カレンダー）、年表、イベント詳細（タスク・ゲスト・お金・メモ）、外部サービス連携

### 1.2 データ・機能
- **イベント**: `id`, `name`, `type`(wedding/funeral/celebration/other), `date`, `event_for`（誰のイベントか）, `location`, `description`, `color`
- **タスク**: フェーズ・期限・担当者（ゲストと紐づけ可能）
- **ゲスト**: 出欠・ご祝儀/香典額
- **収支**: 収入/支出・カテゴリ
- **年表**: 縦軸＝年、横軸＝人（自分/子供/家族など）の切り替え、年代フィルタ
- **外部サービス**: 喪服・着物レンタル、式場予約、引き出物・仏具など（カテゴリ分け表示、仮URL）

### 1.3 ギャップ（ユースケースとの対応）
| ユースケース | 現状 | 不足 |
|-------------|------|------|
| 子供の20歳までに決めることの年表＋通知 | 年表は「過去の出来事」中心。子供向けマイルストーン（七五三・入学等）の**テンプレ**や**締切・通知**の設計が弱い | 子供用テンプレ、通知設計、横軸＝「年齢」の年表 |
| 結婚式までにやることの年表管理 | 結婚式イベント＋タスク・ゲスト・予算はある | 本番日からの**逆算タイムライン**（○ヶ月前で何をする）が画面として弱い |
| 葬式でやることを年表管理 | 葬儀イベント型はあるが、**急な発生**と**短い期間でやることリスト**に最適化されていない | 葬儀用テンプレ、緊急モード、関係者連絡・手続きの優先度 |
| イベント提携（写真館・喪服等） | 外部サービス一覧あり | イベント種別ごとの**おすすめ提携**、写真館の明示、コンテキスト連携が弱い |

## 2. 新規事業・プロダクトとしての評価

### 2.1 強み
- **ターゲットが明確**: ライフイベントを「プロジェクト化」したい層（結婚・出産・葬儀・子供の節目）に刺さるコンセプト
- **年表＋タスク＋お金＋ゲスト**が一画面にまとまっており、一つのイベントを「最後まで見える化」できる
- **event_for** により「誰のイベントか」でフィルタ・年表表示ができ、家族単位の管理の土台がある
- **冠婚葬祭の提携**は差別化と収益化の接点になる

### 2.2 改善すべき点
- **入口の設計**: 「何のイベントを管理したいか」で最初に選ばせる（子供20歳まで / 結婚式 / 葬儀 / その他）と、テンプレと通知が効く
- **時間軸の見せ方**: 「本番日までの残り」と「○ヶ月前・○週間前」のマイルストーンを前面に
- **葬儀**: 発生が予測困難なため、「葬儀用チェックリスト」を常に準備しておき、イベント作成＝「葬儀モード開始」で即テンプレを適用する設計が有効
- **子供向け**: 生年月日から「20歳までの年齢軸」で何をいつまでに決めるか（保険・教育費・行事など）をデフォルトで提示

### 2.3 収益化の方向性（提携ページと整合）
- 写真館・喪服レンタル・式場・仏具などへの**アフィリエート／紹介料**
- 提携サービスを**イベント種別・フェーズ**に紐づけて「今やるべきこと」の横で露出（例: 結婚式タスク「ドレス決め」の横にドレスレンタル提携）

## 3. UI/UX 設計の提案

### 3.1 情報アーキテクチャの整理

```
[ホーム]
  ├ メイン CTA: 「次のイベントまであと○日」＋ 直近タスク
  ├ クイックアクション: イベント種別で追加（子供の節目 / 結婚式 / 葬儀 / その他）
  └ 今日・今週の通知バッジ

[年表]（横軸＝時間の「人生タイムライン」）
  ├ モード切替: 「子供20歳まで」 / 「結婚式まで」 / 「葬儀」 / 「すべて」
  ├ 「子供20歳まで」: 横軸＝子供の年齢(0〜20)、縦＝やるべきこと・マイルストーン・通知
  ├ 「結婚式まで」: 横軸＝本番日からの逆算（12ヶ月前〜当日）、縦＝タスク・予算・提携
  └ 「葬儀」: 横軸＝日（当日〜四十九日等）、縦＝手続き・連絡・会場・提携（喪服等）

[イベント詳細]
  ├ 概要: 残り日数・進捗・次のアクション
  ├ タスク: フェーズ別＋期限＋担当（現状維持＋「推奨提携」をタスク横に表示）
  ├ ゲスト / お金: 現状維持
  └ メモ: 実装＋写真・書類リンク（将来）

[外部サービス（提携）]
  ├ イベント種別でフィルタ: 結婚式 / 葬儀 / 七五三・成人式 / その他
  ├ 写真館を明示したカテゴリ追加
  ├ 現在開いているイベントに紐づく「おすすめ」を上部に表示
  └ 利用履歴・割引コード（将来）
```

### 3.2 画面単位のUX改善

| 画面 | 改善案 |
|------|--------|
| **ホーム** | イベント種別ごとのカード（子供/結婚/葬儀）を並べ、タップで該当年表 or イベント作成に遷移。通知は「期限○日」「未着手タスク」をバッジで表示。 |
| **年表** | 上部で「子供の20歳まで」「結婚式まで」「葬儀」をタブ化。子供モードは「年齢軸」と「やるべきことテンプレ」をデフォルト表示。結婚式は「○ヶ月前」ラベルを軸に表示。 |
| **イベント詳細** | 最初のブロックに「次にやること」1つを大きく表示。タスクカードに「関連する提携サービス」を小さく表示（写真館・レンタル等）。 |
| **設定/外部サービス** | サービス名に「写真館」「喪服レンタル」等を明示。イベント種別タブで絞り込み。現在のイベントに紐づくおすすめを上部に。 |

### 3.3 通知設計（UXの要）
- **プッシュ**: タスク期限（3日前・1日前・当日）、「○ヶ月前になりました」マイルストーン
- **アプリ内**: ホームに「期限が近いタスク」「未アサインのタスク」を常に表示
- **子供20歳まで**: 年齢ベースのリマインダー（例: 「七五三は3歳の○月頃」）

### 3.4 アクセシビリティ・トーン
- 葬儀モードは色を落とし、文言を「ご準備」「ご手続き」など配慮した表現に
- 重要度の高いタスクはアイコン・色・余白で視覚的に優先度を明確に

### 3.5 スマホメイン時の年表の軸設計（縦軸 vs 横軸）

**前提**: 利用の中心がスマホである場合、**時間を横軸にした年表は見づらくなりがち**です。

| 軸の取り方 | スマホでの体感 | 理由 |
|------------|----------------|------|
| **横軸＝時間**（左が過去→右が未来） | ❌ 見づらい・操作しづらい | 幅が狭いため横スクロールが多くなり、一覧性が落ちる。時間範囲が長い（例: 子供0〜20歳）と特に厳しい。 |
| **縦軸＝時間**（上が過去→下が未来、または逆） | ✅ 使いやすい | 縦スクロールはスマホの主操作。画面幅いっぱいでカードやリストを表示でき、読む・タップしやすい。 |

**推奨**:
- **デフォルトは「縦軸＝時間」**にする。年や年齢をセクション見出しにし、その下にイベント・タスクを縦に並べ、**縦スクロールで追う**構成がスマホ向き。
- 「子供の20歳まで」も、**横軸＝年齢（0〜20歳）のガント風**ではなく、**「0歳」「1歳」…「20歳」を縦のセクション**にして、各セクション内でやるべきことをリスト表示する形にすると、スマホで扱いやすい。
- 横軸の年表（例: ガント風・タイムライン風）は**タブレットやWeb**向けのオプションや、**短い期間だけ横に広げる**補助表示として残すのはあり。

**現状の年表画面**: 縦型（年ごとにセクションで縦スクロール）と横型（人物ごとにスワイプ）がある。スマホメインなら**縦型をデフォルト**にし、横型は「人物切り替え」のUIとして残す程度でよい。

## 4. AI 機能の提案

### 4.1 実装優先度が高いもの

| 機能 | 内容 | 価値 |
|------|------|------|
| **タスク・マイルストーン自動提案** | イベント種別＋本番日を入力すると、「結婚式なら12ヶ月前〜当日」の標準タスクを自動生成。子供イベントなら年齢ベースで「七五三」「入学」等を提案。 | ゼロからリストを作る手間を削減し、抜け漏れ防止。 |
| **期限のスマート提案** | タスクの「○ヶ月前」「○週間前」を本番日から逆算して自動設定。ユーザーは微調整だけ。 | 計画の一貫性と通知の精度向上。 |
| **提携サービスのコンテキスト表示** | タスク内容（例: 「着物レンタル」「写真撮影」）を解釈し、該当する外部サービスをイベント詳細や年表の該当タスク横に表示。 | 収益化＋ユーザーは「今やることに合った」サービスに出会える。 |

### 4.2 中期的に検討したいもの

| 機能 | 内容 | 価値 |
|------|------|------|
| **自然言語でのイベント・タスク追加** | 「来年6月に結婚式。式場もまだ」→ イベント作成＋「式場予約」等のタスクと提携リンクを提案。 | 入力のハードルを下げ、初回体験を改善。 |
| **予算の目安・アラート** | イベント種別と規模（ゲスト数等）から支出の目安を提示。実績が目安を大きく上回る/下回る場合に通知。 | 予算感の共有とコスト意識の向上。 |
| **葬儀用チェックリストの自動生成** | 「葬儀」イベント作成時に、関係者連絡・会場・僧侶・会食・香典返し・法要など、一般的な手順に沿ったタスクを自動生成。 | 動揺している状況でも漏れを減らす。 |

### 4.3 データ活用・将来像

| 機能 | 内容 | 価値 |
|------|------|------|
| **複数イベントの重複・負荷の可視化** | 同じ時期に結婚式と引越しが重なっている場合など、負荷が高い時期をハイライトし、タスクの前倒しを提案。 | 現実的な計画づくり。 |
| **過去イベントからの学習** | 同じ種別の過去イベントのタスク・予算を匿名集計し、「同じようなイベントでは平均○件のタスク」「平均予算○円」を参考表示。 | 経験の少ないイベントでも見通しが立ちやすい。 |

### 4.4 技術的な置き場所
- **テンプレ生成・タスク提案**: ルールベース＋必要に応じてLLM（イベント種別・日付・event_for を入力に短いプロンプトでタスクリスト生成）
- **自然言語入力**: チャット風UIで1メッセージを受け取り、LLMでイベント名・日付・タスクを構造化してAPIで保存
- **提携のコンテキスト表示**: タスクの `title` / `category` と提携サービスのタグをマッチング（キーワード or 埋め込み類似度）

## 5. 実装の優先順位（提案）

1. **フェーズ1（短期）**  
   - 年表に「子供20歳まで」「結婚式まで」「葬儀」の**モード/タブ**を追加し、表示軸を変える。  
   - 結婚式モードで「○ヶ月前」ラベルを表示。  
   - 外部サービスに**写真館**を追加し、**イベント種別フィルタ**を実装。

2. **フェーズ2（中期）**  
   - イベント種別×本番日（または子供の生年月日）から**タスク・マイルストーンのテンプレ自動生成**。  
   - **通知**（タスク期限・マイルストーン）の設計と実装。  
   - タスク横に**関連提携**を表示。

3. **フェーズ3（中長期）**  
   - **AIタスク提案**（LLM or ルール）の組み込み。  
   - **自然言語でのイベント・タスク追加**。  
   - 葬儀用チェックリストの自動生成と「葬儀モード」のUI最適化。

## 6. まとめ

- **ターゲット**（ライフイベントを管理したいユーザー）と**3大ユースケース**（子供20歳まで／結婚式／葬儀）は明確で、現状の年表・タスク・ゲスト・お金の土台の上にのせやすい。
- **UI/UX**は「時間軸の見せ方」（年齢軸・逆算軸・葬儀の日軸）と「イベント種別ごとの入口」をはっきりさせ、**通知**で「やるべきこと」を逃さない設計にするとよい。
- **AI**はまずテンプレと期限提案で負荷を下げ、その上で自然言語入力・予算目安・提携のコンテキスト表示を足すと、差別化と収益化の両方に効く。
- **提携ページ**は、イベント種別・フェーズ・タスクと紐づけて「今必要なサービス」を出すことで、ユーザー体験と収益の両方を伸ばしやすい。

このドキュメントを「プロダクト評価とUI/UX・AIロードマップ」のたたき台として使うことを想定しています。実装に落とす際は、フェーズ1から順に小さくリリースし、利用状況を見ながらフェーズ2・3を調整することを推奨します。
