# ビジネスプラン — ライフイベント管理サービス（LifePath）

**作成日**: 2025年3月  
**目的**: 新規事業としての全体像・市場・実行計画の整理

---

## 1. エグゼクティブサマリー

**LifePath** は、結婚式・**子供の節目**・葬儀などのライフイベントを「年表×タスク×お金×ゲスト」で一括管理し、抜け漏れなく準備できるアプリである。  
**ターゲット（中核）**: **子育て世代**（30〜40代を中心に、子供が0〜20歳前後のファミリー）。七五三・入学・誕生日・成人式など「子供の年齢に合わせてやるべきこと」を年表で把握し、家族と共有したい層に刺さる設計とする。  
**サブターゲット**:  
- **冠婚葬祭**（結婚式・葬儀・七五三・成人式・法事など慶弔全般）。  
- **終末・遺産整理**（終活、遺品整理、相続準備、エンディングノート的な「やるべきこと」の整理）。  
入口の認知は子育て世代に置きつつ、アプリ内では冠婚葬祭と終末・遺産整理まで一つの年表・タスクで扱う。  
**収益**: 提携サービス（写真館・衣装レンタル・式場・葬儀・遺品整理等）への紹介料を中核とし、プレミアム課金で継続収益を追加する。  
**差別化**: 「子供の20歳まで」を軸にした年表モードと、タスク・タイミングに紐づいた提携で、**子育て世代が「いつ何をすればいいか」を逃さない**プラットフォームにしつつ、**冠婚葬祭から終末・遺産整理まで**を同じアプリでカバーする。

## 2. 市場・ターゲット

### 2.1 ターゲットの置き方：子育て世代を中核に

**メインターゲット**: **子育て世代**（子供がいる、またはこれから持つ予定の世帯。年齢的には 30〜40代が中心で、子供は 0歳〜20歳前後を想定）。

- **なぜ刺さるか**: 七五三・入園・入学・誕生日・成人式など「子供の年齢に合わせて決めること・準備すること」が多く、**「いつまでに何をすればいいか」を年表で一覧したい**ニーズが強い。共働きも多く、夫婦や祖父母との**共有**ニーズも高い。
- **訴求の軸**: 「子供の20歳までにやるべきことを、年表で逃さず管理」「家族と一緒に準備を進められる」。
- **サブターゲット**は**冠婚葬祭**（結婚・葬儀・慶弔全般）と**終末・遺産整理**（終活・遺品整理・相続準備）まで狙う。同じアプリ内で年表・タスクとして扱い、提携（式場・葬儀社・遺品整理業者等）にもつなぐ。初期の認知・リーチは子育て世代向けを優先する。

### 2.2 市場規模（日本・概算）

| 領域 | 市場規模（年） | 備考 |
|------|----------------|------|
| 七五三・成人式・入学等 | 数百億円規模 | 衣装・写真・食事。**子育て世代の主要イベント** |
| 出産・子育て準備 | 関連サービス含め大きい | 写真・用品・保険・習い事等 |
| 結婚式・ウェディング | 約2.5兆円 | 式場・衣装・写真等。**冠婚葬祭** |
| 葬儀・法事 | 約1.6兆円 | 葬儀社・斎場・仏具等。**冠婚葬祭** |
| 終活・遺品整理・相続 | 数百億円規模以上 | 遺品整理業者・相続相談・終活サービス。**終末・遺産整理** |

※ 当サービスは「管理ツール」であり、上記市場の「入口」として紹介料を得るポジション。**まずは子育て関連を優先**し、続いて冠婚葬祭・終末・遺産整理の提携を広げる想定。

### 2.3 ターゲットユーザー（優先順）

| 優先度 | セグメント | ペルソナ例 | ニーズ | 主なイベント・用途 |
|--------|------------|------------|--------|---------------------|
| **1** | **子育てファミリー** | 30〜40代、子供0〜20歳、共働き多い | 七五三・入学・誕生日等を**年齢軸の年表**で把握し、家族と共有したい | 子供の節目、出産、入園・入学 |
| **2** | **冠婚葬祭** | 結婚準備・葬儀対応・法事等 | 結婚式は逆算タスク、葬儀は急なチェックリスト、法事は日程・会食まで一括管理 | 結婚式、葬儀、七五三・成人式、法事・法要 |
| **3** | **終末・遺産整理** | 終活中の本人・家族、相続準備・遺品整理が必要な層 | 「いつかやる」をリスト化し、家族と共有。遺品整理・相続の手順をタスクで漏れなく | 終活、遺品整理、相続準備、エンディングノート |
| 4 | その他 | 還暦・移転・単発の慶弔等 | 単発イベントのタスク・予算管理 | その他 |

### 2.4 競合との位置づけ

| 競合・代替 | 特徴 | 当サービスとの違い |
|------------|------|---------------------|
| 一般TODO・メモアプリ | 汎用タスク管理 | ライフイベント専用の年表・テンプレ・提携連携がない |
| 結婚式アプリ | 結婚式に特化 | 子供・葬儀・終活など複数イベントを一つの年表で見られない |
| 葬儀・終活系情報サイト | 情報・業者一覧 | タスク・スケジュール・家族共有が弱く、子育て〜終末までつながらない |
| 手帳・カレンダー | 日付管理 | イベント種別ごとの逆算・マイルストーン・提携に弱い |

**ポジション**: **子育て世代**向けに「子供の年齢に合わせたやるべきこと」を年表で一括管理し、写真館・衣装レンタル等の提携に自然につなぐ。**冠婚葬祭**（結婚・葬儀・慶弔）と**終末・遺産整理**（終活・遺品整理・相続）まで同じアプリで扱う「人生の節目から終末まで」のプラットフォーム。

## 3. 提供価値（Value Proposition）

**子育て世代に刺さる価値**を前面に：

1. **子供の「いつまでに何をするか」を逃さない**: 七五三・入園・入学・誕生日・成人式などを**年齢軸の年表**で可視化し、通知でリマインド。「子供の20歳まで」でやるべきことが一覧できる。
2. **家族と一本で共有**: タスク・ゲスト・予算をイベント単位でまとめ、パートナーや祖父母と共有しやすくする（リンク・LINE招待）。
3. **タイミングに合った提携**: 年表・タスクの文脈に合わせて、写真館・着物レンタル・入学用品等を紹介し、準備の負荷を下げる。
4. **記録としての年表**: 終了したイベントも年表に残し、子供の成長の履歴として振り返れる。

**冠婚葬祭**（結婚式・葬儀・法事等）は同じアプリでイベント種別ごとのテンプレ・提携で対応。**終末・遺産整理**は「終活」「遺品整理」「相続準備」などをタスク・年表で管理し、家族と共有したり、提携（遺品整理業者・相続相談等）につなげる設計とする。

## 4. ゴー・トゥ・マーケット（GTM）

### 4.1 チャネル（子育て世代にリーチする優先）

| アプリストア（iOS/Android） | ASOで「七五三 準備」「子供 年表」「ライフイベント 管理」等を意識した説明・スクリーンショット | 高 |
| 提携先との相互導線 | 写真館・衣装レンタル・入学用品等の提携先サイト・店頭で「準備はLifePathで」とQR・リンク | 中 |

| 広告 | キーワード（七五三 準備 アプリ、入学 チェックリスト等）の検索・SNS広告（予算に応じて） | 低〜中 |

### 4.2 ローンチ〜初期の流れ

1. **クローズドβ**: **子育て世代の知人・ママ友・パパ友**を中心に利用開始し、UI・通知・「子供の年表」の使い勝手を検証。
2. **提携2〜5社と契約**: **写真館・七五三・入学関連の衣装レンタル**を優先。その後、**冠婚葬祭**（式場・葬儀社・仏具等）、**終末・遺産整理**（遺品整理・相続相談等）を順次追加。
3. **アプリ公開**: ストア申請、ASOは**子育てキーワード**を意識。コンテンツで「七五三・入学の準備」を訴求。
4. **計測の整備**: イベント作成数（子供関連の割合）、提携クリック/成約、課金（導入後）をダッシュボードで把握。

## 5. コスト構造

### 5.1 固定費・準固定費（月額イメージ）

| 項目 | 月額（目安） | 備考 |
|------|--------------|------|
| インフラ（Supabase・ホスティング等） | 1〜5万円 | ユーザー数に応じてスケール |
| 決済手数料 | 課金の3〜4% | 売上に連動 |
| ドメイン・ツール類 | 0.5〜1万円 | メール・分析等 |
| 開発・運用（自社） | 人件費・外注費 | 体制による |

### 5.2 変動費

- 提携報酬: 成約時にのみ発生（CPA）。売上＝報酬のため実質粗利100%に近い。
- 広告費: 実施する場合のみ。CACを計測し、LTVの2割以下等の目安で管理。

### 5.3 損益のイメージ（月次・例）

| 項目 | Phase 1（例） | Phase 2（例） |

## 6. 体制・ロードマップ（概要）

### 6.1 体制

- **開発**: 現状のExpo + Supabaseをベースに、通知・課金・提携計測を順次追加。
- **提携**: 営業またはパートナー担当が2〜5社と契約・折衝。
- **マーケ**: コンテンツ・ASO・必要に応じて広告。

### 6.2 マイルストーン（製品との対応）

| 時期 | 製品 | 事業 |
|------|------|------|
| 0〜3ヶ月 | 年表モード（子供/結婚/葬儀）、提携導線・計測 | 提携2〜5社、β検証 |
| 4〜6ヶ月 | 通知、AIテンプレ提案、提携のコンテキスト表示 | ストア本公開、**冠婚葬祭**提携拡大 |
| 7〜12ヶ月 | プレミアム課金、共有機能、**終末・遺産整理**用テンプレ・導線 | 課金MRR目標、**遺産整理・終活**提携検討 |
| 13ヶ月〜 | B2B検討、法人向け、終活・相続コンテンツ強化 | 収益多様化 |

## 7. リスクと前提条件

| リスク | 内容 | 対応 |
|--------|------|------|
| 市場リスク | ニーズが想定より小さい | 利用ログ・インタビューで検証し、ペルソナや訴求を調整 |
| 競合リスク | 大手が類似サービスを投入 | イベント種別・年表・提携の深さで差別化、スピードで先行 |
| 提携リスク | 成約単価・条件が厳しい | 複数社・複数カテゴリで分散、自社リストの価値を高める |
| 法務・プライバシー | 個人情報・利用規約・特定商取引 | 規約・プライバシーポリシーを整備し、必要なら法務チェック |
| 技術リスク | スケール・障害 | インフラの選定・監視、バックアップ・ロールバック手順の整備 |

**前提条件**:  
- 提携先が紹介・成約をレポートまたはパラメータで紐づけてくれること。  
- アプリストアの審査に通過し、運用コストを許容できること。

## 9. まとめ

- **何をやるか**: ライフイベント（**子供の節目**・結婚・葬儀等）を年表とタスクで管理し、提携サービスへ導線を出して紹介料を得る。あわせてプレミアム課金で継続収益を確保する。
- **誰に**: **メインは子育て世代。**サブターゲットに**冠婚葬祭**（結婚・葬儀・慶弔全般）と**終末・遺産整理**（終活・遺品整理・相続準備）を置く。「子供の20歳まで」から冠婚葬祭、終末・遺産整理までを同じアプリの年表・タスクで扱う。
- **どう儲けるか**: 提携CPA（子育て→冠婚葬祭→遺産整理の順で提携を広げる）を主収益に、課金を副収益に。計測しつつ単価・成約率・継続率を改善する。
- **次のアクション**: 子育て世代向けコンテンツ・キーワードの整理、写真館・七五三・入学関連の提携候補リストアップ。中期で冠婚葬祭・終末・遺産整理のテンプレと提携候補を整理し、計測仕様と Phase 0 の目標数値を確定する。
