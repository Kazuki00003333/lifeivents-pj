# 仮想ユーザーペルソナとUI/UX評価の仕組み

**目的**: LifePathを「誰の目線で」「何を基準に」評価するかを固定し、**ペルソナに基づくUI/UX評価**を再現可能にする。評価は人間が実施することを前提とし、AIによる補助（ペルソナ役のフィードバック）も行えるようにする。

---

## 1. 仮想ユーザーペルソナ（3名）

ビジネスプランのターゲット（子育て世代・冠婚葬祭・終末・遺産整理）に合わせて3ペルソナを定義する。評価時は「この人の立場で使ってみる」と想定する。

---

### ペルソナA：田中 美咲（たなか みさき）— 子育て世代

| 項目 | 内容 |
|------|------|
| **年齢・家族** | 36歳。夫（38歳）、長女（5歳・年中）、長男（2歳）。共働き。 |
| **職業・生活** | 会社員（事務）。平日は保育園送迎、土日は子供中心。スマホは主に通勤・寝る前。 |
| **ニーズ** | 七五三（長女・2年後）、入学準備（1年後）を「忘れない・家族と分担したい」。年齢で「いつ何をするか」が一覧で見たい。 |
| **技術** | アプリは普段から使う。長文入力は苦手。写真・LINE・カレンダーは日常的。 |
| **評価時の着眼点** | 年表がスマホで縦に読めるか、パパと共有しやすいか、七五三・入学の「やるべきこと」がすぐ見つかるか。 |

---

### ペルソナB：佐藤 健一（さとう けんいち）— 冠婚葬祭

| 項目 | 内容 |
|------|------|
| **年齢・家族** | 29歳。婚約者（28歳）。半年後に結婚式予定。実家は地方、二人は都市部在住。 |
| **職業・生活** | エンジニア。リモート多め。式の準備は主に週末にまとめてやりたい。 |
| **ニーズ** | 式場・招待・衣装・予算を「本番日から逆算」で管理したい。親・婚約者とタスクを共有したい。急な葬儀（親族）にも対応できるツールがあればよい。 |
| **技術** | デジタルツールは得意。ただし式の準備は初めてで、何をいつやるかは不案内。 |
| **評価時の着眼点** | 「○ヶ月前」のタスクが分かるか、招待リンクやLINEで共有しやすいか、結婚式と葬儀の両方が同じアプリで扱えるか。 |

---

### ペルソナC：山本 和子（やまもと かずこ）— 終末・遺産整理

| 項目 | 内容 |
|------|------|
| **年齢・家族** | 58歳。夫（60歳）、子供は独立。義母の介護が一段落し、自分の終活・夫の相続準備を「いつか」リストにしたい。 |
| **職業・生活** | パート勤務。スマホは連絡・検索中心。長いフォーム入力は避けたい。 |
| **ニーズ** | 終活・遺品整理・相続の「やるべきこと」を漏れなくリスト化し、夫や子供と共有したい。急な葬儀時のチェックリストも欲しい。 |
| **技術** | 基本的なアプリ操作はできる。複雑な画面遷移や小さいボタンは負担。 |
| **評価時の着眼点** | 終活・遺産整理の入口が分かるか、タスクがシンプルに読めるか、家族共有が簡単か。文字サイズ・余白・トーンが配慮されているか。 |

## 2. 評価の軸（評価観点）

以下の観点で、各ペルソナごとに「その人ならどう感じるか」を評価する。スコアは 1〜5（1=とても悪い、5=とても良い）または 良い / 普通 / 要改善 でよい。

| # | 観点 | 内容（評価時の問い） |
|---|------|----------------------|
| 1 | **発見しやすさ** | やりたいこと（年表・イベント追加・共有・提携）に、迷わずたどり着けるか。 |
| 2 | **タスクの分かりやすさ** | やるべきこと・期限・担当が一目で分かるか。縦スクロールで追いやすいか（スマホ）。 |
| 3 | **家族共有** | 招待リンク・LINE共有が簡単か。共有された側が「参加する」までがスムーズか。 |
| 4 | **スマホでの操作性** | 片手・縦持ちで操作しやすいか。タップ領域・文字サイズ・スクロール量は適切か。 |
| 5 | **AIの透明性・コントロール** | （AI機能がある場合）何がAIか分かるか、提案を選んで追加・編集・取り消しができるか。[AI_ERA_UI_UX_BEST_PRACTICES.md](./AI_ERA_UI_UX_BEST_PRACTICES.md) に沿っているか。 |
| 6 | **ペルソナ固有のゴール** | そのペルソナの主目的（例: 美咲＝七五三・入学の一覧、健一＝結婚の逆算、和子＝終活リスト）が達成しやすいか。 |
| 7 | **エラー・フォールバック** | 失敗時に次のアクションが分かるか。AIが使えないときの手動の逃げ道があるか。 |
| 8 | **トーン・配慮** | 葬儀・終活などセンシティブな文脈で、文言・色・余白が配慮されているか。 |

---

## 3. シナリオ（タスク）— ペルソナ別

評価時に「この順で操作する」と決めておくと、結果を揃えやすい。以下は一例。

### ペルソナA（田中美咲）

1. アプリを開き、**長女の七五三**用のイベントを追加する（または「子供の節目」から選ぶ）。
2. **年表**で、七五三までの「やるべきこと」が年齢や時期で見えるか確認する。
3. **夫に共有**する（招待リンク or LINE）。共有画面まで迷わず行けるか。
4. タスクを1件「完了」にし、**次のやるべきこと**が分かるか確認する。

### ペルソナB（佐藤健一）

1. **結婚式**のイベントを追加し、本番日を入力する。
2. 「○ヶ月前」の**逆算タスク**（式場、招待状など）が表示されるか確認する。
3. **婚約者を招待**し、相手が参加した想定でタスク一覧を見る。
4. **外部サービス**（式場・衣装）への導線が、タスクやイベントに紐づいて見えるか確認する。

### ペルソナC（山本和子）

1. **終活**または**遺産整理**に近いイベント／タスクの**入口**を探す（メニュー・ホーム・年表）。
2. 終活用の「やるべきこと」を**1件追加**する（テンプレ or 手動）。操作が重くないか。
3. **夫や子供に共有**する流れが分かるか。
4. 画面の**読みやすさ**（文字サイズ・余白・コントラスト）を確認する。

## 4. 評価の進め方（運用）

1. **評価対象を決める**: 対象バージョン・画面（例: 年表・イベント詳細・共有フロー）を明記する。
2. **ペルソナを1人選ぶ**: 今回の評価は「美咲」「健一」「和子」のどれの目線で行うか決める。
3. **シナリオを実行**: 上記のタスクを順にやり、**実際に操作**する（実機 or シミュレータ）。
4. **評価シートを記入**: 下記のテンプレートに、各観点のスコア・気づき・改善案を書く。
5. **複数ペルソナで回す**: 別の日に別ペルソナで同じシナリオを回し、ペルソナごとの差を出す。
6. **（任意）AIで補助**: 下記「5. AIを使ったペルソナ評価」のプロンプトを使い、画面説明や仕様を入力して「そのペルソナならどう感じるか」のコメントを得る。人間の評価を代替ではなく**補足**として使う。

---

- **ペルソナ**: 田中美咲 / 佐藤健一 / 山本和子 のいずれか
- **評価者**: （名前 or チーム）

### スコア（1=とても悪い 〜 5=とても良い）

| 観点 | スコア | メモ・気づき |

## 6. AIを使ったペルソナ評価（補助）

評価者以外に「そのペルソナの立場でコメントする」役をAIに任せる場合のプロンプト例。**画面のスクショ説明・仕様・ワイヤー要約**を [入力] に貼り、AIに「このペルソナならどう感じるか」を短く出させる。人間の評価は代替せず、**気づきの補足**として使う。

```
あなたは【ペルソナ名】です。【以下、ペルソナの要約を貼る】

例: 36歳、共働き、長女5歳・長男2歳。七五三と入学準備を「年表で一覧」「家族と共有」したい。スマホメイン、長文入力は苦手。

以下の【アプリの画面説明 or 仕様 or ワイヤー要約】を見て、「自分（このペルソナ）だったらどう感じるか」を200字程度で教えてください。特に (1) やりたいことにすぐたどり着けるか (2) スマホで読み・操作しやすいか (3) 家族と共有しやすそうか、の3点に触れてください。

【入力】
（ここに画面説明・仕様・ワイヤーを貼る）
```

- **ペルソナ名・要約**は上記「1. 仮想ユーザーペルソナ」の該当者をコピーして使う。
- 出力は評価シートの「メモ・気づき」や「改善提案」のたたき台にする。

## 7. まとめ

| 項目 | 内容 |
|------|------|
| **ペルソナ** | 3名（子育て＝田中美咲、冠婚葬祭＝佐藤健一、終末・遺産整理＝山本和子）。属性・ニーズ・評価時の着眼点を固定。 |
| **評価の軸** | 8観点（発見、タスクの分かりやすさ、共有、スマホ操作性、AIの透明性・コントロール、ペルソナゴール、エラー・フォールバック、トーン）。 |
| **シナリオ** | ペルソナ別のタスクリストで「何をやるか」を統一。 |
| **運用** | 対象バージョン・画面を決め → ペルソナを選ぶ → シナリオ実行 → 評価シート記入。必要に応じてAIでペルソナ目線のコメントを補足。 |
| **テンプレート** | 上記「5. 評価記入テンプレート」をコピーして毎回記入。 |

この仕組みを使えば、**仮想ユーザーペルソナを立てたうえで、同じ観点・同じシナリオでUI/UXを評価**し、今回作成するアプリ（LifePath）の評価を繰り返し行える。
